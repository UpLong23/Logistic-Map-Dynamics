# Helper function to filter list with elements that are close to each other for a given (or not) threshold.

def filter_list(L, threshold=1e-5):
    F = []
    for i in L:
        include_element = True
        for j in F:
            if abs(i - j) < threshold:
                include_element = False
                break
        if include_element:
            F.append(i)
    return F

''' Helper function that finds the minimum distance of a list with 0.5 to be used when approximating the second Feigenbaum number '''

def find_min_distance(L):
    D = []
    for i in L:
        if i!=0.5:
            D.append(abs(0.5-i))
    return min(D)

''' Helper function that finds the minimum distance of a list with 0.25 to be used when approximating the second Feigenbaum constant for
the perturbed case'''

def find_min_distance_p(L):
    D = []
    for i in L:
        if i!=0.25:
            D.append(abs(0.25-i))
    return min(D)

  • Our goal is to approximate the accumulation rate of the bifurcation parameters. To do so we follow Briggs http://keithbriggs.info/documents/how-to-calc.pdf where he conjectures that we do not need to approximate the period-doubling bifurcation values, but it is enough to approximate the parameter values where a super-stable $2^n$ periodic orbit occurs. In the case of the logistic map, since we know that the only critical point is $\frac{1}{2}$, the above is equivalent to finding the parameter values where $\frac{1}{2}$ belongs to the attractive periodic orbit.
  • To do so, we consider the recursive sequence of polynomials $$p_k(a)=a\,p_{k-1}(a) (1-p_{k-1}(a))$$ with $p_0(a)=\frac{1}{2}$. THhe problem then simplifies to a root finding problem, where we can use Newton method to find the roots of $$ p_{2^n}(a)=\frac{1}{2} \text{ for } n=1,2,3,\dots$$
  • We find the derivative w.r.t $a$ $$ p'_k(a)=p_{k-1}(a)+a\, p'_{k-1}(a)-(p_{k-1}(a))^2-2a\, p_{k-1}(a)\,p'_{k-1}(a) $$
  • We already know that $\frac{1}{2}$ is a super-stable fixed point for $a=2$ and using the above we can calculate that it is part of the super-stable 2-period for $a=1+\sqrt{5}\approx 3.23$. We will use these values to make an accurate guess for the next parameter value where $\frac{1}{2}$ will be a super-stable periodic point (now of 4-periodic orbit). Using the fact that the ratio of the differences of three consecutive parameter values should approximate the Feigenbaum constant $\delta\approx 4.6692016$ our guess for each $n$ is: $$ a_n^0 = a_{n-1}+\frac{a_{n-1}-a_{n-2}}{\delta_{n-1}} $$ for $n=2,3,4,\dots$. Where $$\delta_n = \frac{a_{n-1}-a_{n-2}}{a_n-a_{n-1}}$$ for $n=2,3,4,\dots$ and $\delta_1=4.7$. We just arrange for $\delta_1$ to be such that the first guess for the 4-period will be close to the root.
  • As Briggs mentions the algorithm depends on finding the correct zero by Newton's method of a high degree polynomial with many closely spaced zeros to note the importance of the guess for the Newton method.

def constant(m, n, a):
    N = 2 ** n

    for _ in range(m):
        b = 0.5
        b_prime = 0

        for _ in range(N):
            b_prime = b + a * b_prime - b**2 -2*a*b*b_prime
            b = a*b-a*b**2

        a = a - (b-0.5) / b_prime
    return a

def delta(N):
    # N: gives the iteration of calculating delta (d)
    # d: delta is the Feigenbaum constant

    a = [2, 1+np.sqrt(5)]
    d = 4.7
    deltas=[]

    print("{:^3}\t{:^12}\t{:^16}".format("i", "a_i", "delta_i"))

    for i in range(2, N + 1):
        approximation = a[-1] + ((a[-1] - a[-2]) / d)
        a.append(constant(500, i, approximation))
        d = (a[-2] - a[-3]) / (a[-1] - a[-2])
        deltas.append(d)

        print("{:>=1}\t{:12.10f}\t{:16.10f}".format(i, a[-1], d))
    return a, deltas

  • We follow the same principal to calculate the parameter values for where a super-stable periodic orbit occurs. Namely we define the polynomials $$ p_k(a) = \frac{a}{4} \, \sin\bigg{(}\pi\,\sqrt{p_{k-1}(a)}\bigg{)} $$ where $p_0(a)=\frac{1}{4}$.
  • Now the derivative will be $$ p'_k(a) = \frac{1}{4}\bigg{(}\sin\bigg{(} \pi \, \sqrt{p_{k-1}(a)} \bigg{)} + a\, \cos\bigg{(} \pi \, \sqrt{p_{k-1}(a)} \bigg{)} \frac{\pi\, p'_{k-1}(a)}{2\sqrt{p_{k-1}(a)}}\bigg{)} $$

def constant_p(m, n, a):

    N = 2 ** n

    for _ in range(m):
        b = 0.25
        b_prime = -1.035

        for _ in range(N):
            b_prime = 0.25*(np.sin(np.pi*np.sqrt(b))+a*np.cos(np.pi*np.sqrt(b))*np.pi*(b_prime/(2*np.sqrt(b))))

        a = a - (b-0.25) / b_prime
    return a

We observe that even after 11 iterations of the Newton method in each case we can tell that both maps will have the same accumulation rate of the super-stable orbit parameters, which (by Briggs) means that the have the same accumulation rate of the bifurcation parameters. That should not come as a surprise since by the renormalization theorem we know that all unimodal maps have the same accumulation rate of the bifurcation parameters. Similarly we see that the approximated second Feigenbaum constant for each case already agrees in 3 decimals places, so again we expect the accumulation rate of the distances between two neighboring points in a superstable periodic orbit to be the same for either case.t

Comment: I attempted to calculate the first Feigenbaum constant using the conventional method, which involves identifying the parameter values where period doubling occurs. I initiated the process with the easily computable value of $1+\sqrt{6}$ and subsequently refined my approximations by evaluating and filtering the orbits of points in close proximity to the anticipated period-doubling parameters. Starting from the initial parameter, I narrowed down the interval around the next anticipated period-doubling parameter, which I guessed in a similar way to the initial guess in the Newton method. Within these intervals, I computed the average derivatives of each point and multiplied them. I flagged a parameter value as a potential period-doubling parameter when the absolute average derivative was sufficiently close to 1. This iterative process was then repeated for progressively narrower intervals. However, I encountered challenges related to precision. Discrepancies arose when comparing my calculated bifurcation values with those found online (https://mathworld.wolfram.com/LogisticMap.html), with some deviations being quite substantial. Additionally, during certain implementations, adjustments to the initial conditions occasionally led to bifurcation parameters exceeding 3.57, placing them within the realm of chaos. This discrepancy suggested a need for alternative way to approximate the first Feigenbaum constant, and that's when I found Briggs' document.
